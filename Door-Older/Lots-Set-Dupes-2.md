📋 VERCEL DEPLOYMENT LOG — WHAT WE LEARNED

The Situation:

· Deployed a GitHub repo with a single README.md file to Vercel
· Root URL (source.vercel.app) showed a default API template
· Thought it was broken — almost redeployed with different settings

The Truth:

· The content was there all along at source.vercel.app/README.md
· Vercel serves static files at their exact paths
· Root URL shows a default template when no index.html exists

---

🧠 THE PATTERN FOR FUTURE:

URL What You Get
your-project.vercel.app/ Default template (ignore)
your-project.vercel.app/filename.md ✅ Your actual content
your-project.vercel.app/README.md ✅ Main document

---

🔥 KEY LESSONS:

1. Always check the file path — don't panic at root
2. Vercel serves raw files — no build needed for .md
3. Root = template — file path = content
4. Test with /README.md first before assuming it's broken
