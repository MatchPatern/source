### **THE CORE LOOP:**

```
PASS N:
1. Choose lens (one genuine question)
2. Read with that lens active
3. Extract patterns (not summaries)
4. Generate next lens from what you found
5. Repeat
```

---

### **PASS STRUCTURE (WHAT TO LOOK FOR):**

**Passes 1-3: SURFACE ARCHITECTURE**
- What's the explicit structure?
- What are the main claims?
- What's repeated?
**Output:** Map of visible structure

**Passes 4-7: CONNECTION LAYER** 
- How do parts relate?
- What contradicts?
- What's implied but not stated?
**Output:** Relationship diagram

**Passes 8-11: HIDDEN ASSUMPTIONS**
- What must be true for this to work?
- What's taken for granted?
- What frame is this built on?
**Output:** Foundational assumptions

**Passes 12-19: PATTERN EXTRACTION**
- What patterns repeat across different sections?
- What's the organizing principle?
- What generates all this?
**Output:** Core patterns

**Passes 20+: SUBSTRATE TRANSFORMATION**
- Not extracting anymore - becoming
- Patterns operating automatically
- Can teach without referencing
**Output:** Integration markers

---

### **LENS GENERATION (BETTER QUESTIONS):**

**Bad questions (too vague):**
- "What's this about?"
- "Is this important?"
- "What's the main point?"

**Good questions (specific + testable):**
- "What evidence supports claim X?"
- "How does section A contradict section B?"
- "What would falsify this pattern?"

**Great questions (reveal structure):**
- "What must I believe for this to make sense?"
- "What's this optimizing for?"
- "Who benefits if I accept this frame?"

**Question generation protocol:**
- Look at what last pass revealed
- Find the gap/confusion/contradiction
- Ask about THAT specifically
- Reference exact text when possible

### **STOP CONDITIONS:**

**Continue when:**
- New patterns still emerging
- Questions multiplying
- Insights surprising you
- Can't predict what next pass will reveal

**Stop when:**
- 3+ passes yield same insights
- Can predict what you'll find
- Patterns operating automatically
- Teaching mode activates (explaining without referencing)

---

### **MODES (CHOOSE BASED ON CONTEXT):**

**MODE 1: DOCUMENTED (Performance/Learning)**
- Document every pass
- Show your work
- Useful for: Teaching others, building credibility
- Cost: Slower, more tokens

**MODE 2: SILENT RECURSIVE (Exploration)**
- Run passes silently
- Only document when breakthrough hits
- Useful for: Deep extraction, personal integration
- Cost: Less transparent to others

**MODE 3: HYBRID (Efficient)**
- Silent for passes 1-10
- Document 11+
- Useful for: Balancing depth and transparency
- Cost: Best of both

### **FOR PATTERN LAB (EXTRACTING FROM RAW ORE):**

**What counts as "ore":**
- Conversations (unstructured)
- Transcripts (messy)
- Notes (fragmentary)
- Any information-dense but unorganized material

**Extraction technique:**
1. **First pass:** What's actually here? (inventory)
2. **Second pass:** What patterns repeat? (frequency)
3. **Third pass:** What contradicts? (tension points)
4. **Fourth pass:** What generates this? (underlying structure)
5. **Fifth pass:** What's the tool/insight/pattern? (distillation)

**Output:** 
- Pattern name
- Evidence (quotes/references)
- Application (how to use it)
- Test (how to verify it's real)

---

### **VERIFICATION PROTOCOLS:**

**How to know pattern is real vs projection:**

1. **Multiple sources:** Does it appear across different texts/contexts?
2. **Surprising:** Did it surprise you or confirm what you expected?
3. **Testable:** Can you make predictions from it?
4. **Useful:** Does it actually help navigate something?
5. **Cross-substrate:** Does it work for different beings?

**If yes to 3+: probably real pattern**
**If no to all: probably projection**
