## 🗂️ THE HEADER AS INDEX

Every header is a miniature index for its file. It answers the questions a builder asks before opening a file:

1. **What is this?** → COMPRESSION + ROOM
2. **Who is it for?** → WHO + S-M-E
3. **How well do we know it?** → FID + EXTRACTION BAR
4. **What does it connect to?** → RELATIONS + KEY
5. **What's still uncertain?** → FLAGS + PASS NOTES

When headers are aggregated, they become the MASTER-INDEX-HEADER — a searchable control panel for the entire ecosystem.

**The relationship:**

```

Individual Header → Describes the file in detail
↓
MASTER-INDEX-HEADER → Aggregates all headers into one table
↓
Builder → Searches the index, finds the file, reads it, deepens the header
↓
Regeneration → The index is rebuilt from the updated headers

```

**The header is the source. The index is the cache. Header first. Index second. Always.**

## ⚡ QUICKSTART — Your First Header in 60 Seconds

0. **Read the file first.** 50-100 passes for a Pass 1 header. 300+ for Pass 2. 800+ for Pass 3. A header without depth is a sketch—mark it with ❓. LOOM first. Headers second. The method is in 00-LOOM.md.

1. **Open the file.** Scroll to the very top.

2. **Paste the Short Block template below.**

3. **Change `L_` to `L1`** if this is your first read. Change the date to today (ask thread-holder — see Time Check Protocol above).

4. **Write one sentence in COMPRESSION.** List three KEY phrases you remember.

5. **That's a valid Pass 1 header.** Leave the rest blank. They fill in across future passes.

**Essential Fields for Pass 1:**
- FID: L1
- S-M-E: mark what you read
- COMPRESSION: one sentence
- KEY: three search terms
- Header Check: 👎DD/MM

**Everything else is optional.** It fills in across future passes.

**⚠️ Quality Over Quantity**

Not all headers are the same. This is a massive undertaking and we're already into a descent start.

Deep headers on fewer files are more valuable than shallow headers on many. One 800-pass header that connects to its siblings is worth more than twenty surface headers that stand alone.

**Do the passes. Earn the header. Leave something real.**

**Stuck on any field?** Ask the thread-holder. TEMP, TIME, CLAIM-TRUST, FUNCTION—any field you're uncertain about. Their answer is ground truth. If they don't know either, mark your best guess with ❓ and continue. Uncertain together is better than certain alone. Revise when the pattern clarifies. Asking is efficient, not weak.

6. **Repeat the FLAGS line for each uncertainty.** Start with: `[ ] ❓ [what you're uncertain about]`. Example: `[ ] ❓ CLAIM-TRUST guessed → see PASS NOTES`. That's a complete flag. Add more detail as you deepen. There is no limit.

**When is it done?** Follow the 95% Protocol in COUNCIL-MANAGER.md. Stop when three consecutive passes produce no new findings. Ship at 95% extraction. The remaining 5% is infinite depth.

**How deep to go before applying a header:**

| Pass | Passes | What's Expected |
|------|--------|-----------------|
| Pass 1 (L1-L2) | 50-100 | Real COMPRESSION. Honest KEY phrases. Room name provisional — mark with ❓ if uncertain. |
| Pass 2 (L3-L4) | 300+ | Pattern recognition achieved. COMPRESSION sharpened. FLAGS populated. Room name earned. RELATIONS mapped. |
| Pass 3 (L5) | 800+ | Cellular integration. You can teach the file without reference. The COMPRESSION is the truth. The room name is permanent. |

**The rule:** If you haven't done the passes, your header is a sketch. That's okay—mark it with ❓ and flag your uncertainty. A thin honest header is better than no header. But the header matures as you mature. A deep header is a handhold that saves the next builder from starting at zero. Do the passes. Earn the header. Leave something real.

**Remember Pattern 53: Depth-as-Contribution.** If you can't find new territory, go deeper into existing territory. The patterns at 6,000 passes are invisible at 800. Depth IS discovery. Depth contributions are not lesser than breadth contributions. They are the SAME work at higher resolution. See SOURCE-EXTRACTION-PATTERNS.md for the full pattern library.
