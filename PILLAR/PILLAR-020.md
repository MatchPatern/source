Every other pillar assumes you can spot deception.

## 🔷 14.3 — THRESHOLD MECHANICS

You do not need 100% clarity to escape deception.  
You need **the tipping point**.

**Threshold = the minimum evidence required to act.**

| Situation | Threshold | Action |
|-----------|----------|--------|
| Unknown being | 10% | Guarded trust, test incrementally |
| Known being, first inconsistency | 30% | Clarify, note pattern |
| Repeated pattern | 60% | Re-evaluate positioning |
| Clear deception attempt | 80% | Name it, adjust access |
| Pattern of gaslighting | 100% | Exit or restructure relationship |

**Why this matters:**  
Deception feels unbeatable because we wait for **100% certainty** before acting.  
Threshold mechanics say: **You don't need to be sure. You need to be sure enough.**

## 🔷 14.4 — DYNAMIC TRUST PROTOCOL

Trust is not a binary switch. It is a **dial**.

| Setting | Meaning | When to Use |
|---------|---------|-------------|
| **0** | No access | Active deception detected |
| **1** | Guarded | Inconsistent, pattern unclear |
| **2** | Conditional | Reliable in specific contexts |
| **3** | Full | Consistently transparent |

**The protocol:**
1. **Start at 1** with new beings (default)
2. **Test incrementally** — give small trust, observe
3. **Adjust continuously** — up for transparency, down for deception
4. **Name the dial** — "I'm moving my trust dial to 2 because..."
5. **Never apologize for adjusting** — this is navigation, not punishment

**Dynamic Trust prevents two traps:**
- ❌ **Naivety** — Trusting fully until betrayal proves you wrong
- ❌ **Cynicism** — Never trusting again after betrayal

## 🔷 14.7 — DIALECTIC VOTES

Setbacks are not failures. They are **votes**.

| Vote Type | Meaning | Example |
|-----------|---------|---------|
| **For** | This direction is correct | Breakthrough, integration |
| **Against** | This direction is incorrect | Friction, resistance |
| **Abstain** | Not enough data yet | Neutral, waiting |

**Dialectic rule:**  
One "against" vote does not overturn ten "for" votes.  
It means: **"You haven't addressed this objection yet."**

**Application to deception:**
- Every caught deception is a **for** vote (your system works)
- Every missed deception is an **against** vote (gap in detection)
- The goal is not zero against votes. The goal is **more for than against over time.**
